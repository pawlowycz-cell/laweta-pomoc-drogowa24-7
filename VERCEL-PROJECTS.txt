Два сайта: INNSER (большой) и визитка laweta-warszawa.net
==========================================================

ВАЖНО: онлайн обновляется из Git. Правки только на рабочем столе
~/Desktop/MINI COD без копирования в репозиторий на сайте не появятся.


1) INNSER — папка репозитория INNSER COD (большой сайт)
-------------------------------------------------------
  Сборка:  npm run prepare:all   или   npm run build
  → sites/innser/dist/

  Vercel:
    Root Directory:  sites/innser
    Build:           npm run build
    Output:          dist
    Конфиг:          sites/innser/vercel.json

  Домены: www.warszawa-laweta.com, warszawa-laweta.com, laweta-pomoc-* → мост на warszawa-laweta.com


2) Визитка https://www.laweta-warszawa.net — папка mini-site/ + корневой vercel.json
-----------------------------------------------------------------------------------
  Исходники в Git:  mini-site/

  Деплой с корня репо (рекомендуется, если в Vercel не трогали Root Directory):
    В корне лежит vercel.json: build копирует mini-site → .vercel-mini-out
    Vercel: Root Directory ПУСТОЙ, Output Directory ПУСТОЙ (берётся из vercel.json)
    Build: подставится из vercel.json (node scripts/vercel-laweta-build.mjs)

  Альтернатива: Root Directory = mini-site, тогда корневой vercel.json не используется;
    Build: npm run build, Output: пусто, конфиг mini-site/vercel.json

  sync с рабочего стола:  npm run sync:mini-in
  Локально:  npm run preview:mini

  Импорт pomoc:  node mini-site/scripts/import-pomoc-html.mjs "<файл>"

  Домены только здесь: www.laweta-warszawa.net, laweta-warszawa.net

  Если проект Vercel всё ещё смотрит на корень репо или старый output:
    в настройках проекта укажите Root Directory = mini-site и сделайте Redeploy.


Проверка «не меняется на сайте»
--------------------------------
  Сделали push в Git? В Vercel прошла новая сборка?
  Откройте laweta-warszawa.net в окне инкогнито (без кеша).
